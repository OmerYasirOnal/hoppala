# Hoppala v1.6 вЂ” Combo UI Polish

- **Status:** Approved (brainstorm 2026-07-05; user approved, autonomous spec в†’ plan в†’ build в†’ review в†’ ship)
- **Spec round:** v1.6 (deepens the v1.5 stomp combo with live feedback)
- **Predecessors:** v1.5 combo (`world.combo`/`comboEndsAt`/`stompBonus`/`stompFx` already in the sim). Adds only presentation + one deterministic sim field.
- **North star:** retention вЂ” a live combo meter + a best-combo brag + escalating audio make chaining stomps *feel* rewarding, pulling players back.

## 1. Context & goal

v1.5 made stomps worth points; v1.6 makes chaining them *exciting*: a persistent on-screen combo meter with a shrinking timer, a "best combo" line on the game-over screen, and a stomp sound that pitches up as the combo grows. No new gameplay вЂ” pure feedback polish over the existing combo state.

## 2. Principles preserved

- **Deterministic core:** the only sim change is `world.maxCombo` (peak combo this run), updated in the existing `registerStomp` вЂ” pure, seeded, daily-fair. Everything else is a UI (`screens.ts`) / audio (`sfx.ts`) adapter reading combo state.
- **Reachability + determinism tests stay green.** World is y-down. Core entry < 60KB gzip. 60fps. No new deps, no assets.
- **UI copy minimal, TR/EN** via `t`; new strings in `STRINGS`. No new online/backend surface.

## 3. Non-goals (v1.6 boundary)

- No screen shake, no per-combo cosmetics/particles, no combo-gated rewards. Deferred.
- No change to combo *logic* (window, bonus, reset) вЂ” that's v1.5, unchanged.
- No persistent best-combo storage/leaderboard (best-combo is a per-run game-over stat only).

## 4. Sim (deterministic)

`src/game/types.ts`: `World` gains `maxCombo: number`.
`src/game/sim.ts`: `createWorld` inits `maxCombo: 0`; `registerStomp` adds `world.maxCombo = Math.max(world.maxCombo, world.combo)` (after computing `world.combo`). Deterministic вЂ” a fixed daily seed replays identical.

## 5. Combo meter (HUD)

A fixed, horizontally-centered element near the top of the play view (its own node вЂ” NOT in the `.hud` score/zone/mute row, to avoid the mute-button clash), hidden by default:
- `createUI` return gains `setCombo(combo: number, fraction: number): void` and `clearCombo(): void`.
- `setCombo` shows the element, sets the text to `Г—{combo}`, and sets a thin timer bar's width to `fraction` (0вЂ“1, how much of `comboWindow` remains). Bigger/warmer at higher combos (font scale + color by combo tier).
- `clearCombo` hides it (CSS opacity transition for a smooth fade).
- `showHud`/new-run clears it.

`main.ts`, each update while playing: `const frac = (world.comboEndsAt - world.time) / TUNING.comboWindow; if (world.combo >= 2 && frac > 0) ui.setCombo(world.combo, frac); else ui.clearCombo();`

## 6. Best-combo on game-over

`createUI`'s `showGameOver` gains a `bestCombo: number` parameter; when `bestCombo >= 2` it renders a line `{t.bestCombo}: Г—{bestCombo}`. `main.ts` passes `world.maxCombo` at game-over (both the instant render and the online rank-upgrade re-render). New i18n key `bestCombo` (tr `En yГјksek combo`, en `Best combo`).

## 7. Escalating stomp SFX

`src/audio/sfx.ts`: `play(name, pitch = 1)` gains an optional pitch multiplier applied to the effect's base + ramp frequency. `main.ts` stops playing the generic `'stomp'` in the event loop (`e !== 'stomp'`) and instead plays a combo-pitched stomp in the `stompFx` feed: `sfx.play('stomp', 1 + Math.min(fx.combo, 8) * 0.1)` вЂ” each chain level a semitone-ish higher, capped. The `'stomp'` haptic still fires via `bridge.onEvent(e)` in the event loop (unchanged).

## 8. Testing

Pure sim test: `maxCombo` tracks the peak (single stomp в†’ 1; chain в†’ 2; a post-window reset stomp keeps `maxCombo` at the earlier peak). Determinism unaffected. The combo meter, game-over line, and pitched SFX are verified by `pnpm build` + a live Brave smoke (chain two stomps в†’ the meter shows `Г—2` with a shrinking bar, the sound pitches up, game-over shows the best-combo line). Existing tests stay green.

## 9. Acceptance criteria

1. Core < 60KB gzip; all existing + new tests green; determinism intact.
2. A combo в‰Ґ2 shows a centered `Г—N` meter with a timer bar that shrinks and fades when the window lapses; it never clashes with the score/zone/mute HUD.
