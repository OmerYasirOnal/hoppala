# Hoppala v1.7 — Juice & Life

- **Status:** Approved (brainstorm 2026-07-05; user approved full scope "Onaylıyorum, spec'i yaz ve kur"). Autonomous spec → plan → build → review → visual-QA → ship.
- **Spec round:** v1.7 (feel/animation polish + in-game menu access)
- **Predecessors:** v1.2 online, v1.3 zones, v1.4 enemies, v1.5 combo, v1.6 combo-UI. This round adds **presentation only** over the existing simulation — plus one small UI/loop addition (pause).
- **North star:** retention. A game that *feels alive* — squashy player, impact dust, themed 3-D-ish platforms, a screen-shake kick on stomps — is a game players return to. Animations are the explicitly-prioritized retention lever this round.

## 1. Context & goal

The simulation is already rich: 5 platform kinds (static/moving/crumbling/spring/phantom) spawn by altitude, enemies stomp, combos chain. But the **rendering is flat** — platforms are plain rounded rectangles, the player has only a tiny velocity-based squash, and impacts have no feedback. v1.7 injects **juice**: motion, particles, depth, and a tactile screen-shake — without touching the deterministic core. It also closes a user-reported UX gap: **there is no way back to the menu during a run** (you must die to leave).

## 2. Principles preserved (bind all changes)

- **Deterministic core is untouched.** `sim.ts`/`spawner.ts`/`rng.ts` get **no new fields and no new logic**. Every effect in this round is cosmetic and lives in the renderer or in `main.ts` composition, driven by `world.time`, `world.player.vy`, existing `SimEvent`s, and `world.stompFx`. Reachability + determinism tests stay green unchanged.
- **World is y-down.** Altitude px = `startY - player.y`; score = `floor(maxAltitude/10) + stompBonus`. Unchanged.
- **Budgets:** core entry < 60KB gzip (currently ~14.3KB — ample headroom), **60fps on a mid-range phone**. No new runtime deps. **No image/audio assets** — everything procedural on Canvas 2D.
- **Interpolation-safe:** `draw(world, alpha)` may be called multiple times per sim step. All animated effects are **parametric by age** (position computed from `world.time - spawnTime`), never per-frame-accumulated, so interpolation and frame-rate changes never drift them. This mirrors the existing `pops` effect.
- **UI copy** minimal + TR/EN via `t`; new strings in `STRINGS`. Code/docs English.

## 3. Architecture

Two seams, both additive:

1. **`src/render/fx.ts` (NEW, pure, DOM-free, unit-tested):** the math of the juice — particle position/alpha by age, screen-shake decay, per-zone/per-kind platform palette, crumble-fragment placement. No canvas, no DOM; pure functions the renderer calls. This keeps the testable logic pure per the architecture rule (rendering stays a thin adapter).
2. **`src/render/renderer.ts` (extended):** consumes `fx.ts`. Gains bounded effect pools (particles, crumble bursts) and a shake state, plus new public methods (`burst`, `shake`, `landPulse`) that `main.ts` calls from its existing `world.events` / `world.stompFx` loop — exactly how `addPop` is already wired. Richer `drawPlatform` and `drawPlayer`.
3. **Pause:** `main.ts` gains a `paused` flag + a pause button; `screens.ts` gains a pause overlay (a `.modal-overlay`, so it inherits the v1.6.x pointer-events fix) and the in-HUD pause button. No sim involvement.

All effect pools are **capped**; when full, the oldest is recycled (no unbounded growth, no GC churn).

## 4. Components

### 4.1 Player squash-&-stretch + blink + land pulse

`drawPlayer` today applies `squash = min(0.18, |vy|/6000)` (wider when moving). Enhance:

- **Directional stretch:** rising fast (vy < 0) → stretch **vertically** (taller, narrower); falling fast → mild vertical stretch too; the current horizontal-widen becomes the *landing* squash. Use `vy` sign to pick the axis. Keep the magnitude cap so the ball never looks broken.
- **Land pulse:** the renderer tracks the previous `player.vy` sign; on a `+→−` flip (the ball just bounced off something) it records `landPulseTime = world.time`. For ~0.18s after, overlay an extra horizontal squash that eases out (`fx.easeOutPulse(age)`) — the classic "splat then recover". Self-contained in the renderer (no sim change).
- **Blink:** eyes close to a thin line briefly on a deterministic cadence derived from `world.time` (e.g. a short blink every ~3.5s). Cosmetic; cadence from `world.time` so it's stable.

### 4.2 Particle system (renderer, bounded, event-fed)

A single capped pool (`MAX_PARTICLES = 120`). Each particle: `{ x0, y0, vx, vy, spawnTime, life, size, color, gravity }`. **Position is parametric by age** — `fx.particleAt(p, age)` returns `{ x, y, alpha }` where `x = x0 + vx*age`, `y = y0 + vy*age + 0.5*gravity*age²`, `alpha = 1 - age/life` (clamped). Culled when `age > life`. No per-frame integration → interpolation-safe.

Public method `renderer.burst(x, y, kind)` pushes a small cluster (4–10 particles) with kind-tuned velocity/color/gravity. Wired from `main.ts`:

| Trigger (already in the event loop) | Burst |
|---|---|
| `SimEvent 'bounce'` | small dust puff at the player's feet (`player.x`, `player.y + playerR`), light/zone-tinted |
| `SimEvent 'spring'` | bigger upward dust + a couple of yellow sparkles |
| `SimEvent 'break'` | brown crumble fragments at the player's feet (the platform that just broke was under the feet) |
| `SimEvent 'boost'` | warm sparkle burst (jetpack ignite) |
| each `world.stompFx` entry | debris burst at `fx.x, fx.y` (already iterated for `addPop`) |
| pickup taken (main.ts detects `pickup.taken` transition, or on `'boost'`) | fold into the `'boost'` burst — no separate hook needed |

Particles draw between platforms and the player (or just above platforms) so they read against the sky.

### 4.3 Platform theming (dimensional + per-zone + per-kind identity)

`drawPlatform` becomes richer, all procedural:

- **Dimensionality:** a top highlight band (lighter) + a 2–3px bottom shadow/edge (darker) on every platform → reads as a 3-D slab, not a flat bar.
- **Per-zone palette:** `fx.platformPalette(zoneIndex, kind)` returns `{ body, top, edge }` derived from the zone at the platform's altitude, so platforms feel native to Meadow vs higher zones (grass-green low, cooler/rockier high). Falls back to the current per-kind base color as the hue seed so identity is preserved.
- **Per-kind identity (sharper than today):**
  - **spring:** a visible compressed coil/spring glyph on top (not just a nub).
  - **crumbling:** cracks + a slightly crumbly top edge; on `broken` transition, a **crumble burst** (§4.2 `'break'`) plus the slab fades/drops for ~0.35s instead of vanishing instantly. Renderer tracks id→breakTime for the drop-fade.
  - **moving:** a subtle motion sheen / side arrows hinting travel direction.
  - **phantom:** the existing fade, refined into a soft shimmer (alpha pulse) so it reads as "ghostly" not "buggy".
- Keep hitbox/`platformH` identical — **visual only**, reachability untouched.

### 4.4 Screen shake (event-fed, decaying, capped)

Renderer holds `shake = { amp, startTime }`. `renderer.shake(amp)` (called from `main.ts` on each `world.stompFx` entry, `amp` scaled by combo and **capped**, e.g. `min(SHAKE_MAX, 2 + combo)`). During `draw`, `fx.shakeOffset(amp, world.time - startTime)` returns a `{ dx, dy }` that oscillates and decays exponentially to 0 (~0.3s). Applied as a translate on the context before drawing the world, reset after. Cosmetic only — does **not** touch `world.cameraY` or the sim. Respects a hard cap so it never disorients. (This is the juice v1.6 explicitly deferred.)

### 4.5 Parallax background decor (deterministic, per-zone)

Between the sky gradient and the platforms, draw a sparse layer of drifting motifs (soft clouds low, cooler wisps high) at deterministic positions scrolled by `camY * parallaxFactor` (like the existing star field). Per-zone motif/opacity from the zone at `maxAltitude`. Purely decorative, low count, cheap. Gives depth so the climb feels like travel.

### 4.6 In-game pause / back-to-menu (UX)

- A **pause button** in the play HUD (top-left, mirroring the top-right mute), shown only while playing.
- Tapping it **pauses**: set `paused = true`, `loop.stop()`, and show a **pause overlay** (`.modal-overlay`) with **Resume** and **Ana menü / Main menu**.
  - **Resume** → `paused = false`, `loop.start()`, hide overlay.
  - **Main menu** → end the run back to the menu (`showMenu(best, …)`), discard the in-progress run (no score submit — same as abandoning), `playing = false`.
- **Interaction with the existing `visibilitychange` handler:** today `document.hidden` → `loop.stop()`, else `loop.start()`. It must **not auto-resume a manually-paused game** when the tab refocuses. Guard: `visibilitychange`'s `loop.start()` only fires when `!paused`. (This also fixes the latent "background the tab mid-run" behavior.)
- New i18n keys: `pause` (tr "Durdur"/aria), `resume` (tr "Devam"), `mainMenu` (tr "Ana menü", en "Main menu"). The pause overlay benefits from the v1.6.x `.modal-overlay { pointer-events: auto }` fix.

## 5. Determinism & purity guarantees

- No change to `sim.ts`, `spawner.ts`, `rng.ts`, `types.ts` sim fields, `zones.ts` logic. (Types file may gain nothing sim-side; `fx.ts` defines its own small effect types.)
- All juice is a **pure function of** `world.time`, `world.player` (read-only), existing `SimEvent`s, and `world.stompFx` (read-only) — never written back.
- Existing determinism + reachability tests pass **unmodified**.

## 6. Testing

- **`fx.ts` is pure and unit-tested** (`test/fx.test.ts`):
  - `particleAt`: position advances by `vx*age`; gravity curves `y`; `alpha` goes 1→0 across `life`; `age > life` ⇒ culled/alpha ≤ 0.
  - `shakeOffset`: magnitude decays monotonically to ~0 by the decay window; `age ≥ window` ⇒ `{0,0}`; never exceeds `amp`.
  - `platformPalette`: deterministic for a given (zoneIndex, kind); returns distinct body/top/edge; stays within the zone hue family.
  - `easeOutPulse`: 1 at age 0, 0 at/after the pulse window, monotonic.
- **Determinism/reachability suites:** unchanged and green (proves the sim wasn't touched).
- **Renderer/pause (visual + integration):** verified by `pnpm build` + a **live Brave/Playwright smoke** — start a run, confirm particles on bounce, themed platforms, screen-shake on a stomp, the pause button pauses (loop halts) and Resume/Main-menu work; screenshot each. Plus the fresh-context adversarial review reads the diff.

## 7. Acceptance criteria

1. Core entry < 60KB gzip; **all existing tests green unmodified** + new `fx.test.ts` green; determinism & reachability intact.
2. The player squashes on landing and stretches when rising; a dust puff appears on bounce; visibly different bursts on spring/break/boost/stomp.
3. Platforms look dimensional and zone-themed; spring/crumbling/moving/phantom are visually distinct; a crumbling platform breaks with fragments + a short drop-fade, not an instant vanish.
4. A stomp produces a capped, quickly-decaying screen shake that never touches the sim camera.
5. A pause button is available during play; it halts the loop and offers Resume + Main-menu; backgrounding the tab does **not** auto-resume a manually-paused run.
6. 60fps maintained on a mid-range phone (effect pools capped; no assets).

## 8. Non-goals / roadmap

- **No** rewarded-revive / ads this round — that is **v1.8** (first ad slot: game-over "watch → continue", stub now, real network on native later; its own spec).
- **No** horizontal/map/parkour gameplay — the user's "map that expands infinitely to the right" idea conflicts with the locked *vertical* jumper identity. Its on-identity interpretation (a **"Journey Map"** visualizing the existing zones as a travelled route) is a **separate v1.9 brainstorm**, not built here.
- No new platform *kinds* or enemy types (visual variety only). No music. No haptics changes.
