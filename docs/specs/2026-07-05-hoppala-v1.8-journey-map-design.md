# Hoppala v1.8 — Journey Map

- **Status:** Approved (brainstorm 2026-07-05; user chose the "winding trail / adventure" layout and approved the design with "Devam"). Autonomous spec → plan → build → review → visual-QA → ship.
- **Spec round:** v1.8 (the on-identity reinterpretation of the user's "map / infinite expansion" idea). The rewarded-revive ad slot moves to v1.9 to keep the numbering clean.
- **Predecessors:** v1.3 introduced the 8 named altitude ZONES + a flat "Zones collection" list screen (`src/ui/zones.ts`). This round upgrades that list into a visual **Journey Map**.
- **North star:** retention. A map that shows *how far you've come and how far there is to climb* — a winding trail of zones with a "you are here" marker and an endless "∞" beyond the last zone — turns abstract progress into an adventure worth continuing. It reinterprets the user's "a map that expands infinitely to the right" as the game's *actual* structure: the endless **vertical** climb through named zones, without touching the locked vertical-jumper identity.

## 1. Context & goal

The 8 zones (`src/game/zones.ts`: Meadow 0m → Cosmos 2500m, each with a colour) are today shown as a flat vertical list in `renderZones` (`src/ui/zones.ts`): a dot + name + "Xm+"/🔒 per row, "Reached: N/8", Close. It works but reads as a checklist, not a journey. v1.8 redesigns that same modal into a **winding trail**: zone nodes zig-zag up a curved path, the trail you've travelled is bright and the rest is faint, a "▲ you are here" marker sits at your highest zone, and an "∞ endless" cap crowns the top. Same data, same entry point — a far more motivating progression view.

## 2. Principles preserved

- **Zero simulation/gameplay impact.** This is a menu overlay only. No changes to `sim.ts`/`spawner.ts`/`rng.ts`/sim types/`zones.ts` logic. Determinism + reachability + all existing tests stay green, unmodified.
- **No backend, no new data.** Uses existing `ZONES`, the persisted `maxZone` (reached count), and the player's all-time `best` meters. The online layer is untouched.
- **Budgets:** core < 60KB gzip; no image/audio assets (procedural inline SVG). No new runtime deps.
- **UI copy** minimal + TR/EN via `t`; new keys in `STRINGS`. Code/docs English.
- **Offline-first:** the map is fully local (no network).

## 3. Architecture

Two seams:

1. **`src/ui/trail.ts` (NEW, pure, DOM-free, unit-tested):** `trailLayout(count, opts)` — the winding-path geometry. Given a node count, returns each node's `{ x, y, side }` on the zig-zag (bottom-to-top). No DOM; pure math, unit-tested. Keeps the testable logic out of the render adapter.
2. **`src/ui/zones.ts` (rewritten):** `renderZones` builds the Journey Map — an inline `<svg>` inside the existing `.modal-overlay .modal`, using `trailLayout` for node positions: a curved trail `<path>` (travelled segment bright, beyond faint/dashed), zone node circles (zone-coloured if reached, dark if locked), text labels, a "▲ you are here" marker at the current node, and an "∞" cap node at the top. Wrapped in a vertically-scrollable container that auto-scrolls to the current node on open. `main.ts`'s `onZones` handler passes the player's `best` into the view; the rest of the view data it already builds.

## 4. Components

### 4.1 Trail geometry (`trail.ts`, pure)

`trailLayout(count: number, opts: { width: number; vSpacing: number; margin: number }): { x: number; y: number; side: 'left' | 'right' }[]`

- Returns `count` nodes, index 0 = the **bottom** of the map (first zone, e.g. Meadow), index `count-1` = the **top** (last node).
- `y` decreases as the index increases (nodes stack upward). `y[0] = (count - 1) * vSpacing + margin`; `y[i] = y[0] - i * vSpacing`.
- `x` alternates: even index → left lane, odd index → right lane, around the centre (`width/2 ± amplitude`), so the path zig-zags. `side` names the lane.
- Pure and deterministic; unit-tested (monotonic y, alternating sides, x within `[margin, width-margin]`).

### 4.2 Journey Map render (`zones.ts`)

The `.modal` contains: a title (`t.journey`), a subtitle (`t.reached: N/total`), a scrollable SVG map, and a Close button.

The SVG (`viewBox="0 0 W H"`, `H = (nodeCount) * vSpacing + 2*margin` where `nodeCount = zones + 1` for the ∞ cap):

- **Trail path:** a `<path>` through all node centres (bottom→top), drawn as a smooth curve (quadratic segments between consecutive nodes). Rendered as **two** overlaid strokes: the **travelled** portion (from the bottom up to and including the current/highest reached node) in a bright solid stroke; the **unexplored** portion (above it) in a faint dashed stroke. This visually encodes progress.
- **Zone nodes:** a `<circle>` per zone at its `trailLayout` position. Reached (`index <= maxZone`) → filled with the zone's colour (`rgb(top)`) + a light ring; locked (`index > maxZone`) → dark fill (`#33405e`) + `🔒`. A `<text>` label beside each node: reached → zone name + "`{meters} m`" (or "başlangıç/start" for Meadow); locked → "`???`".
- **You-are-here marker:** at the current node (`index = min(maxZone, zones-1)`), a distinct marker (a "▲" glyph or a pulsing highlighted ring) + a label "`{t.youAreHere} · {best} m`". This is the emotional anchor.
- **∞ cap node:** the top node (above Cosmos) is an "∞" glyph + `t.climbForever` — the endless climb. Only meaningfully lit once Cosmos is reached, otherwise shown faint.

`W` ≈ 300–340 (fits the modal width); `vSpacing` ≈ 64; `margin` ≈ 40. Colours/labels come from the existing `ZonesView.rows` (already carries `key/name/meters/color/reached`).

### 4.3 Data flow

`ZonesView` gains `best: number` (the player's all-time best meters, for the you-are-here label). `main.ts`'s existing `onZones` handler already builds `rows` (from `ZONES` + `maxZone`) and `reachedCount`; it additionally passes `best`. No other call site changes. The current-zone index is derived in `renderZones` from `reachedCount - 1` (equivalently `maxZone`), clamped to `[0, zones-1]`.

### 4.4 i18n

New keys in `STRINGS.tr`/`STRINGS.en`: `journey` (tr "Yolculuk", en "Journey"), `youAreHere` (tr "buradasın", en "you are here"), `climbForever` (tr "sonsuza dek tırman", en "climb forever"), `start` (tr "başlangıç", en "start"). Reuse existing `reached`, `close`, and the `zone_*` names. The menu button (`main.ts` `showMenu`) retitles from "🗺️ {zones}" to "🗺️ {journey}".

### 4.5 Scroll & entry

The SVG sits in a `div` with `max-height` (e.g. `60vh`) and `overflow-y: auto`. On open, `renderZones` scrolls that container so the current node is centred/visible (compute the current node's `y`, set `scrollTop`). Entry is unchanged: the menu's map button opens this overlay (a `.modal-overlay`, inheriting the v1.6.x `pointer-events:auto` fix).

## 5. Testing

- **`trail.ts` unit tests** (`test/trail.test.ts`): `trailLayout` returns the requested count; `y` strictly decreases with index (nodes go up); `side` alternates left/right; every `x` and `y` lies within the expected bounds; deterministic for the same inputs.
- **Determinism/reachability/existing suites:** unchanged and green (proves nothing gameplay was touched).
- **Render (visual):** verified by `pnpm build` + a live Brave/Playwright smoke — open the map from the menu, confirm the winding trail renders, reached zones are coloured + named, locked are "???"+🔒, the "you are here" marker sits at the current zone with the best value, the ∞ cap shows, and the container auto-scrolls to the current node; screenshot it. Plus the fresh-context adversarial review reads the diff.

## 6. Acceptance criteria

1. Core < 60KB gzip; all existing tests green unmodified + new `trail.test.ts` green; determinism & reachability intact; zero gameplay/sim change.
2. The Zones menu button opens a **winding-trail** Journey Map (not the old flat list): zone nodes zig-zag up a curved path.
3. Reached zones are zone-coloured + named with their metre threshold; locked zones are dark "???"+🔒; the count "Reached: N/total" shows.
4. A "you are here" marker sits at the player's highest zone, labelled with their best metres; the travelled trail is visually distinct from the unexplored trail; an "∞ endless" cap crowns the top.
5. The map scrolls and auto-scrolls to the current node on open; Close returns to the menu; TR/EN parity for all new copy.

## 7. Non-goals / roadmap

- **No** per-zone bests/stats, no tap-to-expand node details, no new zones, no gameplay/spawn change, no backend — presentation only.
- **No** horizontal/free-roam map or literal rightward-infinite gameplay — that remains off-identity; the endless **vertical** climb (the "∞" cap) is the on-identity expression of "infinite expansion".
- **Deferred to v1.9:** the rewarded-revive ad slot (first monetization step; retention-safe stub now, native AdMob later).
