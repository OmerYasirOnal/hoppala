# Hoppala v1.1 Feature Pack — Design Spec

**Owner:** Ömer Yasir Önal · **Date:** 2026-07-02 · **Status:** approved design, pre-implementation

> Sub-project A of the App Store expansion (decomposition approved 2026-07-02):
> **A) this feature pack** (web + native-shared TS core) → **B) iOS packaging + App Store
> Connect** (Capacitor + official Haptics plugin + a custom ~80-line Swift Game Center
> bridge — its own spec/plan cycle after A ships). Builds on
> `2026-07-02-hoppala-design.md`; all of its constraints (pure sim, reachability
> invariant, budgets, TR/EN copy rules) continue to bind.

## 1. Goal

Deepen the game itself (new hazard, new power-up, a daily shared board) and leave a
clean native-bridge seam so sub-project B can attach Game Center + haptics without
touching the core. Everything here ships to the **web** first as v0.2.0 — web players
get all of it except the native-only integrations.

## 2. Features

### 2.1 Phantom platforms (new hazard)

- A platform kind `phantom` that fades in/out on a per-platform cycle: visible for
  `PHANTOM_ON = 1.6s`, invisible for `PHANTOM_OFF = 0.9s`, phase-offset per platform
  from the seeded RNG. While invisible it does not collide (and renders at 0; during
  the 0.25s fade edges it renders at partial alpha but collision follows the binary
  on/off state — collidable exactly when in the ON window).
- Appears from **5000 px altitude (~500 m)**; `difficultyAt` gains a monotonic
  `phantomRatio` ramping 0 → 0.25 by 15000 px.
- **Invariant preserved by construction:** phantoms are spawned as EXTRA platforms
  alongside the main chain — the main chain (the existing `nextPlatform` sequence)
  stays 100% solid kinds, so the reachability guarantee and its tests are untouched.
  Spawner emits: main-chain platform as today, then with probability `phantomRatio`
  one additional phantom placed at a vertical offset of 0.35–0.65 × the last gap above
  the previous main platform, horizontal position uniform (phantoms may be anywhere;
  they are opportunity/hazard, never the guarantee).

### 2.2 Jetpack boost (power-up)

- A pickup that spawns centered **on top of** a main-chain static platform with
  probability `boostRatio = 0.03` from 2000 px altitude (constant, not ramped).
- Collecting it (circle overlap with the player) sets `player.boostT = 1.2s`: while
  `boostT > 0`, vertical velocity is driven to `BOOST_VY = -1400 px/s` (excess above
  normal bounce), gravity and platform collision are OFF; `boostT` decreases with dt;
  when it hits 0, normal physics resume (falling → normal collision).
- Emits a new `SimEvent` `'boost'` (sfx blip now; haptic in sub-project B).
- Pickups are part of `World` state (`pickups: Pickup[]`), culled like platforms.

### 2.3 Daily mode

- Menu gains a mode choice: **Serbest / Günlük** (Free / Daily).
- Daily seed derives from the LOCAL date: `core/daily.ts` exposes
  `dateKey(d: Date): string` (`YYYY-MM-DD` local), `dayNumber(d: Date): number`
  (days since 2026-01-01 local, day 1 = 2026-01-01) and
  `dailySeed(d: Date): number` (FNV-1a hash of `hoppala:<dateKey>`). Same local day
  → identical board for everyone in that timezone-day (Wordle convention).
- Daily runs are replayable within the day. Share text in daily mode becomes
  `Hoppala #<dayNumber> — <score> m! https://hoppala.vercel.app`; free mode keeps the
  existing text.
- Storage stays key `hoppala:v1`, **additive**: `dailyBest?: { key: string; score: number }`
  (overwritten when the local day changes and a new daily run beats 0). Free-mode
  `best` unchanged. Old saves load fine (field optional).
- HUD shows a small `#<dayNumber>` badge in daily mode; game-over screen shows the
  daily best for today when in daily mode.

### 2.4 Native bridge seam (for sub-project B)

- New `src/platform/bridge.ts`:

```ts
export type GameMode = 'free' | 'daily';
export interface PlatformBridge {
  onEvent(event: SimEvent): void;                 // haptics hook
  submitScore(score: number, mode: GameMode): void; // Game Center hook
  showLeaderboard?(): void;                        // optional native UI
}
export const bridge: PlatformBridge;               // no-op on web
export function setBridge(b: PlatformBridge): void; // Capacitor layer installs its own
```

- `main.ts` routes every consumed sim event through `bridge.onEvent(...)` and calls
  `bridge.submitScore(...)` at game over. Web default is a frozen no-op object. The
  core (`game/`, `core/`) never imports the bridge — only `main.ts` does.

## 3. Out of scope (sub-project B)

Capacitor project, Swift Game Center plugin, Haptics wiring, App Store Connect record,
TestFlight, review submission, store assets. Also out: accounts, Android, ads.

## 4. Testing

- `core/daily.ts`: same-day stability, cross-day difference, dayNumber arithmetic,
  seed determinism (pure functions, no `Date.now` inside — callers pass `new Date()`).
- Spawner: main chain remains all-solid (phantom never appears in the chain sequence);
  `phantomRatio` monotonic + capped; phantom extra-platforms land within the viewport;
  boost pickups only on static main-chain platforms.
- Sim: phantom collides only in its ON window (test both phases at a fixed `world.time`);
  boost drives vy and disables collision for exactly `boostT`, then physics resume;
  `'boost'` event emitted once per pickup; determinism test still green with the new
  content (seeded replay).
- Storage: additive roundtrip incl. `dailyBest`; legacy save without `dailyBest` loads.
- Budgets unchanged: bundle < 60KB gzip (expect ~6KB), suite green, `pnpm build` clean.
