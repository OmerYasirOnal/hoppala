# Hoppala v1.5 — Stomp Combo + Score Pop

- **Status:** Approved (brainstorm 2026-07-05; user pre-approved spec → plan → workflow build → review → ship)
- **Spec round:** v1.5 (retention feedback for the v1.4 enemies)
- **Predecessors:** v1.4 in-climb enemies (the `'stomp'` event this builds on). Reuses the deterministic sim + the renderer.
- **North star:** retention. Making stomps *matter* (score bonus) and *feel great* (score pop) rewards engaging with monsters and climbing the leaderboard.

## 1. Context & goal

v1.4 added stompable monsters. v1.5 rewards chaining them: consecutive stomps within a short window build a **combo** whose multiplier grows the **score bonus** each stomp adds to the leaderboard score, and each kill spawns a floating **"+N ×C"** score pop. Hunting and chain-stomping monsters now boosts your rank — a reason to engage — while altitude still dominates the score.

## 2. Principles preserved

- **Deterministic core:** the combo + score bonus live in the pure seeded sim (`sim.ts`/`types.ts`), so a fixed daily seed replays byte-identically and the daily leaderboard is fair. The score pop is a **cosmetic renderer effect** (non-deterministic timing is fine — it never touches score/state).
- **Reachability invariant** untouched. **World is y-down.** Core entry < 60KB gzip. 60fps. No new deps, no image assets, no i18n (pops are numbers).
- **Score = climb + kills:** `score = floor(maxAltitude / 10) + stompBonus`. The bonus is modest so altitude stays dominant; it stays under the online rules' 100000 cap.
- **Event/update boundary:** per-step stomp data is consumed in the composition layer's UPDATE (like `world.events`), NOT read once in render — so pops aren't missed during frame catch-up.

## 3. Non-goals (v1.5 boundary)

- No persistent on-screen combo counter/bar (the pop shows the combo). Deferred.
- No escalating combo SFX / per-combo cosmetics / screen shake. Deferred.
- No change to enemy spawning, collision, or the stomp mechanic itself (v1.4).
- No new online surface — the combined score flows through the existing leaderboard submit unchanged.
- No combo for anything but stomps (platform bounces / springs don't combo).

## 4. Combo + score bonus (pure sim)

### 4.1 State (`src/game/types.ts`)

`World` gains:
```ts
  combo: number;        // current stomp-chain length (0 when inactive)
  stompBonus: number;   // accumulated bonus meters added to the score this run
  comboEndsAt: number;  // world.time by which the next stomp must land to extend the chain
  stompFx: StompFx[];   // this step's stomps (transient, cleared each step) — for the score pop
```
```ts
export interface StompFx { x: number; y: number; bonus: number; combo: number; }
```
`TUNING` gains `stompBonus: 3` (bonus meters per combo level) and `comboWindow: 1.5` (seconds).

### 4.2 Logic (`src/game/sim.ts`)

- `createWorld` inits `combo: 0, stompBonus: 0, comboEndsAt: 0, stompFx: []`.
- `step` clears `world.stompFx = []` at the top (alongside `world.events = []`).
- On EACH kill that emits `'stomp'` (both the foot-sweep stomp AND the boost-plow), after `e.dead = true` and pushing `'stomp'`, apply the combo:
  ```ts
  world.combo = world.time <= world.comboEndsAt ? world.combo + 1 : 1;
  world.comboEndsAt = world.time + TUNING.comboWindow;
  const bonus = TUNING.stompBonus * world.combo;
  world.stompBonus += bonus;
  world.stompFx.push({ x: e.x, y: e.y, bonus, combo: world.combo });
  ```
  (Multiple kills in one step — e.g. a boost plowing through a cluster — chain correctly: `world.time` is constant within a step, so the second kill sees `time <= comboEndsAt` just set and increments again.)

The combo naturally lapses: a stomp landing after `comboEndsAt` restarts the chain at 1. Nothing else resets it mid-run (the display treats `world.time > comboEndsAt` as inactive).

## 5. Score + score pop

### 5.1 Score (`src/main.ts`)

`meters()` becomes `Math.floor(world.maxAltitude / 10) + world.stompBonus`. This flows unchanged into the HUD score, game-over, best, and the leaderboard/cloud submit (the bonus is part of the score everywhere).

### 5.2 Score pop (renderer + composition)

- The renderer owns a pop list (closure state). New API `addPop(x: number, y: number, bonus: number, combo: number, time: number): void` pushes a pop with `spawnTime = time`.
- `main.ts`'s update, after the existing `for (const e of world.events)` fan-out, feeds this step's stomps to the renderer: `for (const fx of world.stompFx) renderer.addPop(fx.x, fx.y, fx.bonus, fx.combo, world.time);` (per step → not missed on catch-up).
- `renderer.draw(world, alpha)` ages pops by `world.time - spawnTime`, drawing a floating `+{bonus}` (append ` ×{combo}` when `combo ≥ 2`) that rises and fades over `POP_DURATION` (~0.8s), larger/brighter for higher combos, culling expired pops. Pops draw in world space (at `x`, `y − camY`, wrap-x) above enemies/player.

## 6. Testing

Pure sim tests (Vitest, no DOM): a single stomp sets `combo === 1` and `stompBonus === TUNING.stompBonus` and pushes one `stompFx`; two quick successive stomps chain (`combo === 2`, `stompBonus === stompBonus·1 + stompBonus·2`); a stomp after the window (force `comboEndsAt` into the past) resets `combo === 1`; determinism holds (`run() === run()` including `stompBonus`). The score-pop render + the `meters()` bonus are verified by `pnpm build` + a live Brave smoke (stomp a monster, see the pop + the score jump). Existing reachability/determinism/collision tests stay green.

## 7. Acceptance criteria

1. Core < 60KB gzip; all existing + new tests green; reachability + determinism intact.
2. A stomp adds `stompBonus × combo` meters to the score; the combo grows on quick successive stomps and lapses after `comboWindow`.
3. Each stomp shows a floating `+N` (with `×C` for combos ≥ 2) at the kill spot that rises and fades.
4. The bonus is deterministic (a fixed daily seed replays identical, kills included) and part of the leaderboard score.
5. Altitude still dominates the score in normal play; the bonus stays under the 100000 cap.

## 8. Roadmap (later)

Persistent combo meter/streak UI, escalating combo SFX + screen-shake, per-combo cosmetics, "best combo" stat on game-over, combo-gated rewards. Ads (rewarded revive) — the monetization endgame.
