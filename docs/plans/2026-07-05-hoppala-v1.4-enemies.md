# Hoppala v1.4 — In-Climb Enemies (Stomp) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add one Doodle-Jump-style monster type inside the climb — stomp its head to kill+bounce, jetpack boost plows through, any other contact ends the run — spawned seeded (daily-compatible), altitude-scaled, without touching the reachability invariant.

**Architecture:** Enemies live in the pure deterministic sim: a new `Enemy` type + `world.enemies`, spawned by `spawner.ts` (a new `enemyRatio`), resolved in `sim.ts` `step` (stomp/boost/death), rendered procedurally (zone-accented) in `renderer.ts`, with a new `'stomp'` `SimEvent` flowing through the existing sfx/haptic fan-out. No new input, no online change, no main.ts change.

**Tech Stack:** Vanilla TypeScript, Canvas 2D, Vitest 4. Reuses the seeded `mulberry32` sim + the v1.3 `zones.ts`.

**Spec:** `docs/specs/2026-07-05-hoppala-v1.4-enemies-design.md`

## Global Constraints

- **Deterministic core:** `sim.ts`/`spawner.ts`/`rng.ts`/`daily.ts`/`zones.ts`/`types.ts` stay DOM-free. Enemies draw from the same seeded RNG stream (runs reproducible; daily works). Adding enemy RNG draws SHIFTS the stream — expected for a new version; determinism *within* v1.4 holds.
- **Reachability invariant** (vertical platform gap ≤ `0.8 × JUMP_HEIGHT`) — UNTOUCHED. Enemies are additive hazards in gaps (never on the guaranteed chain platform's landing spot). The existing reachability tests must stay green.
- **Core entry payload < 60KB gzip** (CI-enforced). 60fps. No image assets (procedural monster). No new dependencies. No i18n (enemies are visual).
- **tsconfig** `strict` + `noUncheckedIndexedAccess`. Test output pristine. Conventional commits. pnpm only.
- **Exact values:** `TUNING.enemyR = 15`, `TUNING.stompVy = -1300`; `enemyRatio = a < 1000 ? 0 : Math.min(0.15, ((a-1000)/8000)*0.15)`; enemy spawns mid-gap at `0.4 + 0.3·rng()` of the last gap.
- **Browser automation uses Brave** (machine rule) for the live smoke.

```ts
describe('enemy spawning', () => {
  const main = (y: number) => ({ id: 1, kind: 'static' as const, x: 200, y, w: 68, baseX: 200, amp: 0, speed: 0, phase: 0, broken: false });

  it('spawns enemies mid-gap (0.4–0.7 of the last gap), inside the viewport, above the platform', () => {
    const rng = mulberry32(5);
    let seen = 0;
    for (let i = 0; i < 800 && seen < 20; i++) {
      const m = main(-150); // last gap 150 (prevY 0 → mainPlat.y -150)
      const { enemy } = spawnExtras(m, 0, 20000, rng, 1000 + i, 2000 + i, 3000 + i) as { enemy: Enemy | null };
      if (!enemy) continue;
      seen++;
      const offset = 0 - enemy.y; // above prevY=0
      expect(offset).toBeGreaterThanOrEqual(150 * 0.4 - 1e-9);
      expect(offset).toBeLessThanOrEqual(150 * 0.7 + 1e-9);
      expect(enemy.y).toBeGreaterThan(m.y); // in the gap, below the next chain platform (not on its landing spot)
      expect(enemy.x - TUNING.enemyR).toBeGreaterThanOrEqual(-1e-9);
      expect(enemy.x + TUNING.enemyR).toBeLessThanOrEqual(TUNING.viewWidth + 1e-9);
      expect(enemy.dead).toBe(false);
    }
    expect(seen).toBeGreaterThan(0);
  });

```

Note (y-down: smaller y = higher): the enemy sits mid-gap between `prevY`=0 and `mainPlat.y`=−150. With offset ∈ [0.4,0.7]·150 = [60,105], `enemy.y = −offset ∈ [−105,−60]`, which is BELOW the next platform (−105 > −150, i.e. lower altitude) yet ABOVE the start (−105 < 0). So `enemy.y > m.y` — the enemy is in the gap the player climbs through, never on the chain platform's landing spot. ✓

```ts
  // enemies: stomp from above (falling) kills + bounces; an active boost plows through;
  // any other contact ends the run.
  for (const e of world.enemies) {
    if (e.dead) continue;
    const dxr = Math.abs(p.x - e.x);
    const dx = Math.min(dxr, TUNING.viewWidth - dxr);
    const dy = p.y - e.y;
    const rr = TUNING.playerR + TUNING.enemyR;
    if (dx * dx + dy * dy > rr * rr) continue;

      p.vy = TUNING.stompVy;

    } else {
      world.over = true;
      world.events.push('gameover');
    }
  }
```

```ts
    // enemies: procedural monster, glow-accented by the current zone, wrap-x like the player
    const zt = ZONES[zoneIndexAt(world.maxAltitude / 10)]!.top;
    for (const e of world.enemies) {
      if (e.dead) continue;
      const ey = e.y - camY;
      for (const ox of [0, -TUNING.viewWidth, TUNING.viewWidth]) {
        const ex = e.x + ox;
        if (ex < -TUNING.enemyR || ex > TUNING.viewWidth + TUNING.enemyR) continue;
        ctx.fillStyle = `rgba(${zt[0]},${zt[1]},${zt[2]},0.5)`; // zone-tinted glow halo
        ctx.beginPath();
        ctx.arc(ex, ey, TUNING.enemyR + 3, 0, Math.PI * 2);
        ctx.fill();
        ctx.fillStyle = '#3a1d5c'; // dark body — high contrast in every zone
        ctx.beginPath();
        ctx.arc(ex, ey, TUNING.enemyR, 0, Math.PI * 2);
        ctx.fill();
        ctx.fillStyle = '#fff';
        ctx.beginPath();
        ctx.arc(ex - 5, ey - 3, 3.2, 0, Math.PI * 2);
        ctx.arc(ex + 5, ey - 3, 3.2, 0, Math.PI * 2);
        ctx.fill();
        ctx.fillStyle = '#1c1c28';
        ctx.beginPath();
        ctx.arc(ex - 5, ey - 3, 1.6, 0, Math.PI * 2);
        ctx.arc(ex + 5, ey - 3, 1.6, 0, Math.PI * 2);
        ctx.fill();
        ctx.strokeStyle = '#ff5a6e'; // angry mouth
        ctx.lineWidth = 2;
        ctx.beginPath();
        ctx.moveTo(ex - 6, ey + 7);
        ctx.lineTo(ex + 6, ey + 7);
        ctx.stroke();
      }
    }
```
