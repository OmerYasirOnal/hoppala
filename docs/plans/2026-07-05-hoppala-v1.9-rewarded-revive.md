# Hoppala v1.9 Rewarded Revive Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a Free-mode "watch an ad → continue this run" revive: a deterministic `revive(world)` that re-enters play via the existing boost, a swappable ad seam (native bridge + web stub), and a game-over button — one revive per run, never in Daily.

**Architecture:** `revive(world)` is a new pure function in `src/game/sim.ts` (reuses `boostT`; unit-tested). The ad is abstracted: `bridge.showRewardedAd?` (native, future) with a web `src/ui/ad.ts` stub fallback. `src/ui/screens.ts` `showGameOver` gains an optional `onRevive` button; `src/main.ts` orchestrates the ad→revive→resume flow, tracks one revive per run, and gates it to Free mode.

**Tech Stack:** Vanilla TypeScript, Canvas 2D, Vite 8, Vitest 4, pnpm.

## Global Constraints

- **Pure, deterministic sim:** `revive` is DOM-free and a deterministic function of world state. Daily mode NEVER revives → daily seeded replay stays byte-identical. Determinism + reachability tests pass unmodified.
- **Reachability:** revive repositions the player mid-view among already-spawned platforms + grants a boost; never fabricates unreachable state.
- **Ads isolated + offline-first:** the ad is behind `bridge.showRewardedAd?` + a web stub; nothing about ads blocks or alters normal play. Anti-cheat unchanged (Firestore rules; cap 100000; monotonic). No real ad network bundled this round.
- **Budgets:** core < 60KB gzip; no image/audio assets; no new runtime deps.
- **UI copy:** new strings in BOTH `STRINGS.tr` and `STRINGS.en`; TR/EN parity. Code/docs English.
- **Commits:** conventional. pnpm. Test: `pnpm test`. Build: `pnpm build`.
- **Existing sim facts (do not change):** `world.over` is the death flag; `step` early-returns when `world.over`. `player.boostT > 0` overrides physics with upward `TUNING.boostVy` flight and makes enemy contact a kill-not-death (invulnerable). Pickups grant `boostT = TUNING.boostDuration` (1.2s). `World` has `cameraY`, `viewHeight`, `maxAltitude`, `stompBonus`, `combo`, `maxCombo`, `events`, `stompFx`, and `player` (`{ x, y, prevX, prevY, vy, boostT }`).

```ts
/**
 * Resume a dead run in place (Free-mode revive). Deterministic: a pure function of world state.
 * Reuses the jetpack boost for a safe re-entry — invulnerable upward flight back into the
 * reachable platform field. Preserves all score-bearing state (maxAltitude, stompBonus, combo).
 */
export function revive(world: World): void {
  world.over = false;
  const p = world.player;
  p.y = world.cameraY + world.viewHeight * 0.5; // mid-view, above the death line
  p.prevY = p.y; // collapse interpolation so the render doesn't streak from the old position
  p.vy = 0;
  p.boostT = TUNING.boostDuration; // invulnerable upward flight clears hazards + reaches platforms
  world.events = []; // don't replay the 'gameover' event
  world.stompFx = [];
}
```

- [ ] **Step 2: Create the web ad stub `src/ui/ad.ts`**

```ts
import { t } from './screens';

/**
 * Web placeholder for a rewarded ad: a short countdown, then resolves true (reward earned).
 * The real network (native AdMob via bridge.showRewardedAd) replaces this later.
 */
export function showRewardedAdStub(root: HTMLElement, seconds = 3): Promise<boolean> {
  return new Promise((resolve) => {
    root.querySelector('.ad-overlay')?.remove();
    const overlay = document.createElement('div');
    overlay.className = 'modal-overlay ad-overlay';
    let left = seconds;
    const draw = (): void => {
      overlay.innerHTML = `<div class="modal ad-modal"><div class="ad-badge">📺 ${t.adPlaceholder}</div><div class="ad-count">${left}</div></div>`;
    };
    draw();
    root.append(overlay);
    const timer = setInterval(() => {
      left -= 1;
      if (left <= 0) {
        clearInterval(timer);
        overlay.remove();
        resolve(true);
      } else {
        draw();
      }
    }, 1000);
  });
}
```

```ts
/** Ad → revive → resume the same run. Guarded against a stale game-over generation. */
async function doRevive(gen: number): Promise<void> {
  if (gen !== overGen || playing) return; // a new run already started
  const rewarded = bridge.showRewardedAd ? await bridge.showRewardedAd() : await showRewardedAdStub(uiRoot);
  if (!rewarded || gen !== overGen || playing) return;
  revivesUsed++;
  revive(world);
  playing = true;
  loop.start();
  ui.showHud();
}
```

## Notes on testing philosophy (read before implementing)

Task 1 is real TDD — `revive` is pure and unit-tested (including a "does not immediately re-die" reachability sanity check). Tasks 2–3 are the ad stub (DOM/timer), the game-over button, and `main.ts` composition, which the project does not unit-test. Their gate is: `pnpm exec tsc --noEmit` clean + the existing tests plus the new revive tests green unmodified + `pnpm build` under budget + a final live Brave/Playwright smoke (die in Free → "Watch & Continue" shows → stub ad → run resumes with a boost, score preserved; Daily never shows it; only one revive per run). Do not invent brittle DOM/timer unit tests to satisfy a TDD checkbox.
