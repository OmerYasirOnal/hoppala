# Hoppala v1.6 — Combo UI Polish Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make chaining stomps feel great — a live centered combo meter with a shrinking timer, a game-over best-combo stat, and a stomp SFX that pitches up with the combo — over the existing v1.5 combo state.

**Architecture:** The only sim change is deterministic `world.maxCombo` (peak combo this run, set in the existing `registerStomp`). Everything else is presentation: `screens.ts` gains a combo meter + a best-combo game-over line; `sfx.ts` gains a pitch multiplier; `main.ts` wires the per-frame meter, the game-over stat, and the combo-pitched stomp sound.

**Tech Stack:** Vanilla TypeScript, Canvas 2D, Web Audio, Vitest 4.

**Spec:** `docs/specs/2026-07-05-hoppala-v1.6-combo-ui-design.md`

## Global Constraints

- **Deterministic core:** the only `sim.ts`/`types.ts` change is `world.maxCombo` (pure, seeded — a fixed daily seed replays identical). UI/audio are thin adapters. Reachability + determinism tests stay green.
- **Core entry < 60KB gzip.** 60fps. No new deps/assets. UI copy in `STRINGS` (tr/en). tsconfig `strict` + `noUncheckedIndexedAccess`. Pristine tests. Conventional commits. pnpm only. Browser automation uses Brave.
- **Exact values:** combo meter shows for `combo >= 2`; SFX pitch `1 + Math.min(combo, 8) * 0.1`; `comboWindow` (1.5s) already in TUNING.

### Task 3: SFX pitch + main wiring

**Interfaces:** Consumes `world.maxCombo`, `world.combo`, `world.comboEndsAt`, `TUNING.comboWindow`, `ui.setCombo`/`clearCombo`, `sfx.play(name, pitch)`.

- [ ] **Step 1: Add a pitch multiplier to `sfx.play`.** In `src/audio/sfx.ts`, change the `play` signature + type + frequency lines.

Return-type annotation: change `play(name: SfxName): void;` to `play(name: SfxName, pitch?: number): void;`.

The `play` implementation — change `play(name) {` to `play(name, pitch = 1) {` and apply `pitch` to the two frequency values:
```ts
      osc.frequency.setValueAtTime(freq * pitch, ac.currentTime);
      osc.frequency.exponentialRampToValueAtTime(Math.max(60, freq * pitch * (name === 'gameover' ? 0.4 : 1.4)), ac.currentTime + dur);
```

- [ ] **Step 2: Wire the combo-pitched stomp + skip the generic stomp** in `src/main.ts`. Change the event loop (currently `if (e !== 'gameover') sfx.play(e);`) to skip `'stomp'`:
```ts
    for (const e of world.events) {
      bridge.onEvent(e);
      if (e !== 'gameover' && e !== 'stomp') sfx.play(e);
    }
    for (const fx of world.stompFx) {
      renderer.addPop(fx.x, fx.y, fx.bonus, fx.combo, world.time);
      sfx.play('stomp', 1 + Math.min(fx.combo, 8) * 0.1);
    }
```
(The `'stomp'` haptic still fires via `bridge.onEvent(e)` in the event loop.)
